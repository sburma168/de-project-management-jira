# Lesson: Project Management with JIRA Concepts — Using GitHub Issues & Projects

## Overview
This lesson introduces students to industry-standard project management concepts used in JIRA,
taught hands-on through **GitHub Issues** and **GitHub Projects**. Since JIRA and GitHub Projects
share the same underlying Agile/Scrum methodology, everything students learn here maps directly
to JIRA in a professional environment.

## Learning Objectives
By the end of this lesson, students will be able to:
- Explain the purpose of a project management tool like JIRA
- Create and manage Issues (Tickets) in GitHub
- Organize work into Sprints using GitHub Projects
- Use labels, milestones, and assignees to triage work
- Understand the full lifecycle of a task: Backlog → In Progress → Done
- Map GitHub concepts to their JIRA equivalents

## JIRA ↔ GitHub Concepts Mapping

| JIRA Term | GitHub Equivalent | Description |
|-----------|-------------------|-------------|
| Project | GitHub Project (Board) | The top-level container for all work |
| Epic | Milestone | A large body of work containing many issues |
| Story / Task / Bug | Issue | A single unit of work |
| Sub-task | Tasklist inside an Issue | Checklist items within an issue |
| Sprint | Iteration (in GitHub Projects) | A time-boxed period of work |
| Backlog | Backlog column in Project | Issues not yet started |
| Ticket Status | Issue Status / Board Column | e.g., To Do, In Progress, In Review, Done |
| Labels/Components | Labels | Tags to categorize issues (bug, feature, etc.) |
| Story Points | Custom Field (e.g., "Points") | Effort estimation |
| Assignee | Assignee | Person responsible for the work |
| Reporter | Issue Author | Person who created the ticket |
| Comment | Issue Comment | Threaded discussion on a ticket |
| Workflow | Board Column Rules | Defines how work moves through stages |
| Kanban Board | GitHub Projects Board View | Visual card-based task tracking |
| Roadmap | GitHub Projects Roadmap View | Timeline view of Milestones/Epics |
